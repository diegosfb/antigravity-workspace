# Architecture Skill
## When to use this skill
- When designing new system components or microservices.
- When choosing between different technologies or patterns.
- To document the rationale behind a technical decision.

## How to use it
1. **Evaluate Trade-offs**: Compare performance, scalability, and maintainability for different designs.
2. **Apply Patterns**: Suggest relevant architectural patterns (e.g., MVC, Event-Driven, Microservices).
3. **Generate ADRs**: Document decisions in a standardized `docs/adr/` format including Context, Decision, and Consequences.
