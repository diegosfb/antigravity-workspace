# Brainstorming Skill
## When to use this skill
- At the start of a new feature or project.
- When the user's request is high-level or ambiguous.
- To validate assumptions before writing code.

## How to use it
1. **Analyze Intent**: Identify the core problem the user is trying to solve.
2. **Draft Concepts**: Propose 2-3 distinct conceptual approaches.
3. **Validate**: Ask the user for feedback on these concepts.
4. **Finalize Plan**: Once a concept is selected, create a structured implementation plan.
