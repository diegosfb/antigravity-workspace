# TDD Skill
## When to use this skill
- When implementing new features or bug fixes using TDD principles.
- To ensure high test coverage and reliable code.

## How to use it
1. **RED**: Write a failing test first. Run it to confirm it fails.
2. **GREEN**: Write the minimal amount of code needed to pass the test. Run it to confirm it passes.
3. **REFACTOR**: Improve the code structure while keeping tests green.
4. **Validation**: Ensure all tests in the suite still pass after each cycle.
