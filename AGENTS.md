# Repo Agent Rules

## Local Policies
- For new features or product requirements, fetch latest context from available MCP servers before coding.
- Prefer small, focused changes and keep file sizes reasonable.
- Avoid adding new top-level docs unless there is no suitable existing location.

## When To Use Which Agent
- Use `ba-analyst` when breaking down features, use cases, or ideas into Jira epics and user stories.
- Use `product-specialist` at project kickoff or when defining new features.
- Use `antigravity-config` when reorganizing skills, rules, workflows, agents, scripts, or project structure.
- Use `code-reviewer` immediately after code changes.
- Use `security-reviewer` before commits affecting auth, data access, or external integrations.
